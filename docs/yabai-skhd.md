# yabai + skhd 利用ガイド

macOS 用タイル型ウインドウマネージャ **yabai** と、キーボードショートカットデーモン **skhd** の利用方法。

## セットアップ

### 1. インストール

```bash
# dotfiles の install.sh で自動インストールされる
./install.sh

# 手動の場合
brew install koekeishiya/formulae/yabai
brew install koekeishiya/formulae/skhd
```

### 2. アクセシビリティ権限の付与（必須）

**システム設定 → プライバシーとセキュリティ → アクセシビリティ** で以下にチェック：

- `/opt/homebrew/bin/yabai`
- `/opt/homebrew/bin/skhd`

権限を付与しないとウインドウ操作が一切効かない。

### 3. サービスの起動・停止

mise タスクとして定義済み。

```bash
# まとめて起動 / 停止 / 再起動
mise run wm:start
mise run wm:stop
mise run wm:restart

# 個別に操作
mise run yabai:start
mise run yabai:stop
mise run yabai:restart

mise run skhd:start
mise run skhd:stop
mise run skhd:restart
```

### 4. 設定ファイル

| ファイル | 用途 |
|---|---|
| `~/.yabairc` | yabai の設定（レイアウト、padding、除外ルール） |
| `~/.skhdrc` | skhd のショートカット定義 |

設定変更後はサービスを再起動する。

### 5. skhd のキーワードと Mac キーの対応

| skhd キーワード | Mac キー |
|---|---|
| `alt` | option (⌥) |
| `cmd` | command (⌘) |
| `shift` | shift (⇧) |
| `ctrl` | control (⌃) ※未使用 |

## レイアウトの仕組み

yabai は **bsp (Binary Space Partitioning)** レイアウトを使用。
ウインドウを開くたびに画面が自動的に分割される。

```
┌──────────┬──────────┐    ┌──────────┬─────┐
│          │          │    │          │  B  │
│    A     │    B     │ →  │    A     ├─────┤
│          │          │    │          │  C  │
└──────────┴──────────┘    └──────────┴─────┘
  2ウインドウ                 3ウインドウ
```

## ショートカット一覧

修飾キーの階層：

| 修飾キー | 操作カテゴリ |
|---|---|
| `⌥` | フォーカス移動（軽い操作） |
| `⇧⌥` | ウインドウ入れ替え |
| `⌘⌥` | ウインドウ移動（warp） |
| `⌘⇧` | サイズ拡大 |
| `⌘⇧⌥` | サイズ縮小 |

### ウインドウフォーカス移動 — `⌥ option` + hjkl

| キー | 動作 |
|---|---|
| `⌥ H` | ← 左のウインドウにフォーカス |
| `⌥ J` | ↓ 下のウインドウにフォーカス |
| `⌥ K` | ↑ 上のウインドウにフォーカス |
| `⌥ L` | → 右のウインドウにフォーカス |

### ウインドウ入れ替え（swap） — `⇧⌥` + hjkl

隣のウインドウと位置を交換する。

| キー | 動作 |
|---|---|
| `⇧⌥ H` | ← 左のウインドウと入れ替え |
| `⇧⌥ J` | ↓ 下のウインドウと入れ替え |
| `⇧⌥ K` | ↑ 上のウインドウと入れ替え |
| `⇧⌥ L` | → 右のウインドウと入れ替え |

### ウインドウ移動（warp） — `⌘⌥` + hjkl

swap と違い、ツリー構造を組み替えて移動する。

| キー | 動作 |
|---|---|
| `⌘⌥ H` | ← 左方向に移動 |
| `⌘⌥ J` | ↓ 下方向に移動 |
| `⌘⌥ K` | ↑ 上方向に移動 |
| `⌘⌥ L` | → 右方向に移動 |

### ウインドウサイズ拡大 — `⌘⇧` + hjkl

| キー | 動作 |
|---|---|
| `⌘⇧ H` | ← 左方向に拡大 |
| `⌘⇧ J` | ↓ 下方向に拡大 |
| `⌘⇧ K` | ↑ 上方向に拡大 |
| `⌘⇧ L` | → 右方向に拡大 |

### ウインドウサイズ縮小 — `⌘⇧⌥` + hjkl

| キー | 動作 |
|---|---|
| `⌘⇧⌥ H` | ← 左方向に縮小 |
| `⌘⇧⌥ J` | ↓ 下方向に縮小 |
| `⌘⇧⌥ K` | ↑ 上方向に縮小 |
| `⌘⇧⌥ L` | → 右方向に縮小 |

### レイアウト操作

| キー | 動作 |
|---|---|
| `⌥ R` | ウインドウを 90° 回転 |
| `⌥ Y` | Y軸でミラー（左右反転） |
| `⌥ X` | X軸でミラー（上下反転） |
| `⌥ E` | 分割方向を切り替え（水平⇔垂直） |
| `⌥ A` | padding / gap の表示切替 |

### フルスクリーン・フロート

| キー | 動作 |
|---|---|
| `⌥ F` | yabai フルスクリーン（タイル内最大化） |
| `⇧⌥ F` | macOS ネイティブフルスクリーン |
| `⌥ T` | フロート切り替え（中央配置） |

### スペース（デスクトップ）操作

| キー | 動作 |
|---|---|
| `⌥ 1`〜`⌥ 6` | スペース 1〜6 にフォーカス移動 |
| `⇧⌥ 1`〜`⇧⌥ 6` | 現在のウインドウをスペース 1〜6 に送る |

## マウス操作

| 操作 | 動作 |
|---|---|
| `fn + 左クリックドラッグ` | ウインドウ移動 |
| `fn + 右クリックドラッグ` | ウインドウリサイズ |

## macOS ショートカットとの競合

skhd はシステムショートカットより優先してキーを捕捉する。以下の macOS 標準ショートカットが上書きされる：

| 上書きされるショートカット | 元の macOS 機能 | 代替手段 |
|---|---|---|
| `⌘⌥ H` | 他のアプリを隠す | メニューバーから操作 |

それ以外の `⌘⇧` / `⌘⇧⌥` の組み合わせは macOS 標準では未使用のため競合しない。

## フローティング除外アプリ

以下のアプリはタイリングされず、通常のフローティングウインドウとして表示される。

- System Settings / システム設定
- Calculator
- Karabiner
- Archive Utility
- Finder（情報ウインドウ等）
- Activity Monitor
- App Store
- 1Password
- OrbStack

除外アプリを追加するには `.yabairc` に以下を追記：

```bash
yabai -m rule --add app="^アプリ名$" manage=off
```

## よく使うコマンド

```bash
# 現在のウインドウ情報を確認
yabai -m query --windows --window

# 全ウインドウ一覧
yabai -m query --windows

# 現在のスペース情報
yabai -m query --spaces --space

# レイアウトを一時的に float に切り替え
yabai -m config layout float

# bsp に戻す
yabai -m config layout bsp
```

## トラブルシューティング

### ショートカットが効かない

1. アクセシビリティ権限を確認
2. `skhd --restart-service` で再起動
3. `skhd -o` でデバッグモード起動（キー入力がログに出る）

### ウインドウが自動タイリングされない

1. `yabai --restart-service` で再起動
2. 対象アプリが除外ルールに含まれていないか `.yabairc` を確認
3. `yabai -m query --windows` で `"is-floating": true` になっていないか確認

### 設定を変更したが反映されない

```bash
mise run wm:restart
```
